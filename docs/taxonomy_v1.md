# Taxonomy v1

The taxonomy is intentionally small for the MVP. It supports future JD-resume matching by creating a shared vocabulary for roles, skills, seniority, and domains.

## Role Groups

- AI Engineer
- AI Researcher
- AI Scientist
- AI Analyst

## Skills

- Python
- SQL
- PyTorch
- TensorFlow
- scikit-learn
- LLM
- RAG
- NLP
- Computer Vision
- MLOps
- Docker
- Kubernetes
- AWS
- GCP
- Azure

## Seniority

- Intern
- Junior
- Mid
- Senior
- Lead
- Principal

## Domains

- Finance
- Healthcare
- Commerce
- Manufacturing
- Mobility
- Gaming
- Education
- Platform
- SaaS

## Connection to JD-Resume Matching

Future matching can use this taxonomy to compare JD requirements with resume evidence. The initial MVP uses rule-based labels so each decision can be inspected before model-based matching is added.
